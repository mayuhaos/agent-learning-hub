# Agent Notes

## Agent Loop

Agent Loop 是 Agent 程序的核心运行方式。它通常包含：接收用户目标、调用模型进行推理、让模型选择下一步动作、执行工具、把工具结果回传给模型，然后继续推理。这个循环会一直运行，直到模型认为已经可以给出最终答案。

一个最小 Agent Loop 不一定复杂。它可以只包含一个模型、几个工具函数、一个循环和停止条件。停止条件很重要，因为模型可能一直尝试调用工具，程序需要用最大步数、超时时间或错误处理来避免无限循环。

## Tool Call

Tool Call 是模型表达“我想调用某个工具”的结构化方式。模型不会真的执行本地函数，它只会返回工具名和参数。程序需要解析这个调用意图，根据工具名找到本地函数，执行函数，再把结果回传给模型。

在 OpenAI Responses API 中，工具调用通常包含 `function_call`、`call_id`、`name` 和 `arguments`。其中 `call_id` 用来把某一次工具调用和对应的工具结果关联起来。

## RAG

RAG 是 Retrieval-Augmented Generation，也就是检索增强生成。它的核心思想是：不要只让模型凭自己的参数记忆回答，而是先从外部资料中检索相关内容，再让模型基于这些资料生成答案。

RAG 很适合课程资料问答、企业知识库、文档助手和研究助手。它能把模型回答限制在可检索资料之内，并且可以通过引用来源来降低幻觉风险。
